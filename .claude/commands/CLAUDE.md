# VTV Commands

Slash commands for AI-assisted development workflows. Run any command by typing `/command-name` in Claude Code.

## Command Reference

### Setup

| Command | Usage | Purpose |
|---------|-------|---------|
| `/init-project` | `/init-project` | Start Docker services, verify prerequisites, check API health |

### Context Loading (output for human)

| Command | Usage | Purpose |
|---------|-------|---------|
| `/prime` | `/prime` | Load full codebase understanding ‚Äî architecture, features, git state, infrastructure |
| `/prime-tools` | `/prime-tools` | Load agent tool system context ‚Äî tool designs, composition chains, docstring standards |

### Planning (output for agent)

| Command | Usage | Purpose |
|---------|-------|---------|
| `/planning` | `/planning add user profiles` | Research and create a detailed implementation plan saved to `plans/` |
| `/plan-template` | `/plan-template` | Output the blank plan template to `plans/template.md` for manual use |

### Execution

| Command | Usage | Purpose |
|---------|-------|---------|
| `/execute` | `/execute plans/user-profiles.md` | Implement a feature from a plan file, then validate |
| `/implement-fix` | `/implement-fix 42` | Implement a bug fix from an RCA document at `docs/rca/issue-42.md` |

### Validation & Review

| Command | Usage | Purpose |
|---------|-------|---------|
| `/validate` | `/validate` | Run all 5 VTV checks: ruff format, ruff check, mypy, pyright, pytest |
| `/review` | `/review app/core/` | Code review against VTV standards (types, logging, architecture, tests) |

### Git

| Command | Usage | Purpose |
|---------|-------|---------|
| `/commit` | `/commit` or `/commit app/core/health.py` | Conventional commit with safety checks and Co-Authored-By footer |

### Investigation

| Command | Usage | Purpose |
|---------|-------|---------|
| `/rca` | `/rca 42` | Root cause analysis for a GitHub issue, saved to `docs/rca/issue-42.md` |

### Autonomous Chain

| Command | Usage | Purpose |
|---------|-------|---------|
| `/end-to-end-feature` | `/end-to-end-feature add health dashboard` | Full lifecycle: prime ‚Üí plan ‚Üí execute ‚Üí validate ‚Üí commit |

## Workflows

### Feature Development (manual steps)
```
/prime                                    # Understand the codebase
/planning add user authentication         # Create the plan
/execute plans/user-authentication.md     # Implement it
/validate                                 # Verify everything passes
/commit                                   # Commit with conventional format
```

### Feature Development (autonomous)
```
/end-to-end-feature add user authentication
```

### Bug Fix
```
/rca 42                    # Investigate and document root cause
/implement-fix 42          # Fix from the RCA document
/validate                  # Verify the fix
/commit                    # Commit with "Fixes #42"
```

### Agent Tool Development
```
/prime-tools                              # Load tool designs and patterns
/planning add obsidian search tool        # Plan with tool-specific sections
/execute plans/obsidian-search-tool.md    # Implement with agent-optimized docstrings
/validate                                 # Verify all checks pass
/commit                                   # Commit
```

### Quick Check
```
/validate        # Run all linting, type checking, and tests
/review app/     # Review code against VTV standards
```

## Output Directories

- `plans/` ‚Äî Implementation plans created by `/planning`
- `docs/rca/` ‚Äî Root cause analysis documents created by `/rca`

## Trust Progression

Follow this progression before using autonomous commands:

1. **Manual prompts** ‚Äî Learn what instructions work
2. **Individual commands** ‚Äî Run `/prime`, `/planning`, `/execute`, `/commit` separately and verify each
3. **Chained commands** ‚Äî Use `/end-to-end-feature` only after trusting each individual command
