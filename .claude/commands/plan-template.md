This command outputs the standard VTV plan template to a file so you can fill it in manually. The template has all the sections that `/execute` expects: feature description, user story, solution approach with alternatives, relevant files with line ranges, step-by-step implementation tasks with per-task validation, testing strategy, logging events, acceptance criteria, and final validation commands.

Use this when you want to write a plan by hand instead of having `/planning` generate one automatically. This is useful when you already know exactly what you want to build and don't need the research phase, or when the feature is unusual enough that automated planning wouldn't capture the nuances. The template serves as a checklist ensuring you don't forget important sections.

After filling in the template, run `/execute plans/{your-plan}.md` to implement it. You can also use this template as a reference when reviewing plans generated by `/planning` to understand what each section is for.
