## Feature Description

Generate a fully-typed TypeScript API client from the FastAPI backend's OpenAPI schema using the already-configured `@hey-api/openapi-ts` toolchain. The `@vtv/sdk` package exists at `cms/packages/sdk/` with its config and dependencies installed, but `src/client/` is empty тАФ no code has been generated yet.

This plan will: (1) save a local copy of the OpenAPI schema for reproducible builds, (2) generate types + SDK functions, (3) wire up authentication via a request interceptor that reuses the existing `authFetch` token extraction pattern, (4) create a configured client instance in the web app, and (5) migrate one existing hand-written client (events-client) as a proof-of-concept to validate the approach end-to-end.

After this plan, future client migrations can follow the same pattern established in the events-client migration.

## Architecture Overview

### Current State
- 9 hand-written API clients in `cms/apps/web/src/lib/*-client.ts`
- Each duplicates: error class, `handleResponse()`, `BASE_URL`, `API_PREFIX`, `authFetch` calls
- Hand-written TypeScript types in `cms/apps/web/src/types/*.ts` (~68 types)
- Backend exposes 47 endpoints and 68 schemas via `/openapi.json`

### Target State
- `@vtv/sdk` generates types + SDK functions from OpenAPI schema
- A single configured client instance in the web app handles auth + base URL
- Hand-written clients are replaced one at a time (this plan does events-client only)
- Hand-written types remain for now тАФ consumers can gradually adopt generated types

## SDK Generation Configuration

### Existing Config (`cms/packages/sdk/openapi-ts.config.ts`)
```typescript
input: "http://localhost:8123/openapi.json"  // Live backend required

```

### What Gets Generated
The `@hey-api/typescript` plugin generates all 68 schema types.
The `@hey-api/sdk` plugin generates typed SDK functions for all 47 endpoints.

Each generated function signature matches the backend operation, e.g.:
- `listEventsApiV1EventsGet({ query: { page, page_size, start_date, end_date } })`
- `createEventApiV1EventsPost({ body: { ... } })`
- `deleteEventApiV1EventsEventIdDelete({ path: { event_id } })`

Operation IDs are auto-generated by FastAPI from the Python function names + route paths.

### Auth Integration Strategy
`@hey-api/client-fetch` supports `client.interceptors.request.use()` тАФ a request interceptor that runs before every request. We will register an interceptor that extracts the JWT token the same way `authFetch` does (dynamic import `auth()` on server, `getSession()` on client) and sets the `Authorization` header.

This means the generated SDK functions automatically include auth without any per-call configuration.

### Task 5: Create Configured SDK Client in Web App
**File:** `cms/apps/web/src/lib/sdk.ts` (create)
**Action:** CREATE

This is the single entry point for all SDK usage in the web app. It creates a `@hey-api/client-fetch` client instance configured with:
- Base URL from environment variable
- Auth interceptor that extracts JWT token (same dual-context pattern as `authFetch`)

```typescript
/**
 * Configured @vtv/sdk client for the VTV web application.
 *
 * Provides a pre-configured client with:
 * - Base URL from NEXT_PUBLIC_AGENT_URL environment variable
 * - Automatic JWT auth token injection via request interceptor
 *
 * Usage:
 *   import "@/lib/sdk";  // Side-effect: configures the global client
 *   import { listEventsApiV1EventsGet } from "@vtv/sdk";
 *   const { data, error } = await listEventsApiV1EventsGet({ query: { page: 1 } });
 */

import { createClient } from "@hey-api/client-fetch";

const client = createClient({
  baseUrl: process.env.NEXT_PUBLIC_AGENT_URL ?? "http://localhost:8123",
});

// Auth interceptor: injects JWT Bearer token into every request.
// Uses the same dual-context pattern as authFetch:
// - Server-side: auth() decodes httpOnly JWT cookie (cheap, no network)
// - Client-side: getSession() fetches from /api/auth/session
client.interceptors.request.use(async (request) => {
  let token: string | undefined;

  if (typeof window === "undefined") {
    const { auth } = await import("../../auth");
    const session = await auth();
    token = session?.accessToken;
  } else {
    const { getSession } = await import("next-auth/react");
    const session = await getSession();
    token = session?.accessToken;
  }

  if (token) {
    request.headers.set("Authorization", `Bearer ${token}`);
  }

  return request;
});

export { client };
```

**CRITICAL тАФ Server/Client Boundary:**
The dynamic `await import()` pattern is mandatory. Static imports of `auth()` (server-only module) would break the client-side bundle. This is the same pattern used in `auth-fetch.ts` (line 29-30 vs 34-35).

### Task 8: Create SDK Wrapper for Events Domain
**File:** `cms/apps/web/src/lib/events-sdk.ts` (create)
**Action:** CREATE

Create a thin adapter that wraps the generated SDK functions with the same API as the current `events-client.ts`. This allows consumers to switch imports without changing their call sites.

The wrapper should:
1. Import the configured client setup (side-effect import of `@/lib/sdk`)
2. Import generated SDK functions from `@vtv/sdk`
3. Re-export functions with the same signatures as `events-client.ts`
4. Map between generated types and existing hand-written types
5. Preserve the `EventsApiError` class for backward compatibility

**Pattern for each function:**
```typescript
import "@/lib/sdk"; // Ensures client is configured with auth + baseUrl
import {
  listEventsApiV1EventsGet,
  createEventApiV1EventsPost,
  getEventApiV1EventsEventIdGet,
  updateEventApiV1EventsEventIdPatch,
  deleteEventApiV1EventsEventIdDelete,
} from "@vtv/sdk";
import type {
  OperationalEvent,
  EventCreate,
  EventUpdate,
  PaginatedEvents,
} from "@/types/event";

/** Error thrown when the events API returns a non-OK response. */
export class EventsApiError extends Error {
  status: number;
  constructor(status: number, message: string) {
    super(message);
    this.name = "EventsApiError";
    this.status = status;
  }
}

/** Fetch paginated events with optional date range filter. */
export async function fetchEvents(params: {
  page?: number;
  page_size?: number;
  start_date?: string;
  end_date?: string;
}): Promise<PaginatedEvents> {
  const { data, error } = await listEventsApiV1EventsGet({
    query: params,
  });
  if (error) {
    throw new EventsApiError(/* extract status */ 500, String(error));
  }
  return data as unknown as PaginatedEvents;
}

// ... same pattern for fetchEvent, createEvent, updateEvent, deleteEvent
```

**IMPORTANT:** The exact function names and parameter shapes depend on what Task 7 discovers in the generated code. The executing agent MUST read the generated `sdk.gen.ts` first, then write this wrapper to match. The code above is illustrative тАФ adapt to actual generated signatures.

**Type compatibility note:** The generated types (e.g., `EventResponse`) may differ slightly from hand-written types (e.g., `OperationalEvent`). Use `as unknown as OperationalEvent` cast initially. If the shapes match (they should since both come from the same backend), this is safe. Future cleanup can remove hand-written types entirely.
