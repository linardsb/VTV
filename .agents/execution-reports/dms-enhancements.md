## Bugs Found During Implementation

| Bug | Root Cause | Fix Applied |
|-----|-----------|-------------|
| `update_document` returns empty tags | Used `DocumentResponse.model_validate(doc)` which doesn't load tags (no ORM relationship) | Changed to `await self.get_document(document_id)` which enriches tags |
| `ingest_document` returns without auto-generated tags | Same `model_validate` issue — tags created after response was built | Changed to `await self.get_document(doc.id)` at return |
| Early return path (no chunks) returns without tags | Same `model_validate` pattern on the early exit | Changed to `await self.get_document(doc.id)` |
| `get_or_create_tag` TOCTOU race condition | Check-then-act pattern: two concurrent requests pass `get_tag_by_name` check, both try `create_tag`, one hits unique constraint | IntegrityError catch + `db.rollback()` + retry `get_tag_by_name` |
| N+1 query in `list_documents` | Called `get_tags_for_document` per document (20 docs = 21 queries) | Added `get_tags_for_documents` batch method with single JOIN query |
| `DocumentNotFoundError` raised for missing tags | Wrong exception type — semantically misleading | Created `TagNotFoundError(NotFoundError)` in exceptions.py |
| `add_tags_to_document` N individual INSERTs | Loop of individual INSERT per tag_id | Batched into single `pg_insert().values(list).on_conflict_do_nothing()` |
| OCR warning missing error context | `except Exception:` without capturing exception | Changed to `except Exception as e:` with `error=str(e)`, `error_type=type(e).__name__` |
| Auto-tag warning missing error context | Same pattern — bare `except Exception:` | Same fix: capture `e`, add structured error fields |
| Status update failure log missing stack trace | `logger.error()` without `exc_info=True` | Added `exc_info=True` to the logger call |
| Lazy import of `DuplicateTagError` in method body | Unnecessarily deferred import for same-package module | Moved to top-level imports |
| `pg_insert(Tag.__table__)` MyPy type error | `__table__` returns `FromClause`, incompatible with `pg_insert` expected type | Switched from pg_insert upsert to IntegrityError catch pattern |

## Divergences from Plan

- **Migration cleanup:** Autogenerated migration included irrelevant pre-existing column changes. Manually cleaned to only include tag and OCR changes.
- **get_or_create_tag pattern:** Plan suggested `pg_insert().on_conflict_do_nothing()` upsert, but MyPy rejected `Tag.__table__` type. Used IntegrityError catch pattern instead (more Pythonic, equally race-safe).
